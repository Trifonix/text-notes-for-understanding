=== Область видимости переменных ===

Переменные могут быть глобальными и локальными. Глобальная - это когда например переменная объявлена наверху, а через функцию к ней идёт обращение. А локальная - это когда переменная объявлена внутри функции. Если определить локальную переменную с тем же именем, которое уже есть у глобальной, то в рамках этой функции будет использоваться локальная переменная. Чтобы поменять значение глобальной переменной изнутри функции используется ключевое слово `global`

name = "Neo"
def func():
	global name
	name = "Smith"
	print(name)			# Вывод: Smith
func()
print(name)				# Вывод: Smith

Ещё есть ключевое слово `nonlocal`, с которым можно изменить значение в нелокальной переменной выше уровнем.

def outer():
    num = 8
    def inner():
        nonlocal num
        num = 42
        print(num)
    inner()			# Вывод: 42
    print(num)		# Вывод: 42
outer()



=== Замыкания ===

Это такая большая функция, которая включает в себя локальное окружение из переменных и внутренних функций. Например, можно создать функцию func, которая при вызове будет передавать аргумент в параметр для внутренней функции и возвращать саму внутреннюю функцию:

def func(first):
	def second(second):
		return first + second
	return second
sum = func(1)
print(sum(10))		# Вывод: 11
print(sum(20))		# Вывод: 21

А при записи возращенной функции в переменную и вызове её, можно будет передать второй аргумент для sum. Таким образом в переменной лежит ссылка на замыкание (функция + окружение).



=== Декораторы ===

Это такая функция-обёртка, которой можно внести изменения в рабочую функцию не меняя её саму. Например, есть функция hello, которая выводит одну строку. Над ней можно объявить декоратор decorator_func. И чтобы применить декоратор в исходной функции, над ней можно написать название декоратора через 'собачку'

def decorator_func(input_func):
    def new_func():
        print('*' * 24)
        input_func()
        print('*' * 24)
    return new_func

@decorator_func
def hello(): print("Hello, World!")
hello()

Так при вызове исходной функции будет вызван декоратор

Ещё декораторы используют для работы с параметрами функции. Например, есть некая функция суммы двух чисел. Можно при её вызове через декоратор внести в неё изменения, получив отдельно переданные в неё аргументы *args, например 2 и -4, и если сумма этих чисел будет меньше 0, то результатом функция вернёт другое значение, например 0, а не сам результат выполнения функции sum

def modify_sum(original_func):
	def result_func(*args):
		result = original_func(*args)
		if result < 0:
			print(f"увы, тут result: {result}") # Вывод: -2
			return 0
		else:
			return result
	return result_func

@modify_sum
def sum(a, b):
	return a + b

print(sum(2, -4))   # Вывод: 0
print(sum(4, 8))    # Вывод: 12

Вообще есть два способа вызвать декоратор. Первый способ - через 'собачку' перед именем исходной функции. Второй спосок - через передачу исходной функции аргументом в декоратор. Например:

def deco():
	pass

def my_func():
	return 42

my_func = deco(my_func)		# Переопределение исходной функции, декоратор принимает функцию

Декоратор - это паттерн программирования, который позволят добавлять в функцию новый функционал не изменяя саму функцию.

Зачем декораторы? Чтобы считать время выполнения функции. Чтобы логировать параметры функции. Чтобы повторить функцию после ошибки. Чтобы поставить ограничение на частоту запусков функции.

На продакшене обычно использую типизацию. Из модуля typing импортируется объект Callable, который можно вызывать.

from typing import Callable
def empty_deco(func: Callable):
	def wrapper():				# обёртка
		res = func()			# вызов самой функции
		return res				# возврат результата вызываемой функции
	return wrapper				# возврат другой функции после переопределения
def my_func():
    return 42
my_func = empty_deco(my_func)
print(my_func())

Простейший пример применения декоратора - это декоратор подсчёта времени выполнения исходной функции.

from typing import Callable
import time
def timer_deco(func: Callable):
    def wrapper():
        start = time.time()
        res = func()
        end = time.time()
        print(f"Функция отработала за: {end-start}")
        return res
    return wrapper
@timer_deco
def hello_func():
    print("Начинаем спать")
    time.sleep(1.5)
    print("Закончили спать")
    return "Hello_World"
print(hello_func())

Также можно сделать декоратор подсчёта времени для функции с параметром. Здесь в качестве параметра будет передаваться время сна. Переметры передаются через 'арги' и 'кварги' в обёртку.

from typing import Callable
import time
def param_timer_deco(func: Callable):
    def wrapper(*args, **kwargs):
        start = time.time()
        res = func(*args, **kwargs)
        end = time.time()
        print(f"Функция работала: {end-start}")
        return res
    return wrapper
@param_timer_deco
def my_func(sleep_time: int):
    time.sleep(sleep_time)
    return 42+42
print(my_func(2))

Декоратор с параметром. Допустим, мы хотим написать декоратор, который будет лимитировать количество вызовов исходной функции. Для этого в декораторе добавляется ещё один уровень вложенности - дополнительная обёртка для параметра лимита.

from typing import Callable
import time
def limit_deco(limit: int):
    def inner(func: Callable):
        def wrapper():
            nonlocal limit
            if 0 == limit:
                print("Лимит исчерпан")
                return 404
            limit -= 1
            start_time = time.time()
            res = func()
            end_time = time.time()
            print(f"Время исполнения: {end_time-start_time}")
            return res
        return wrapper
    return inner
@limit_deco(2)
def my_func():
    time.sleep(.2)
    return 200
print(my_func())
print(my_func())
print(my_func())

При использовании декоратора не так просто вывести докстринг исходной функции. Для получения докстринга и имени исходной функции подключается отдельный декоратор wraps из модуля functools. Для включения этот wraps подключается через 'собачку' перед объявлением функции обёртки внутри декоратора.
from typing import Callable
from functools import wraps
def deco(func: Callable):
    @wraps(func)
    def wrapper():
        """Менее важный докстринг"""
        print("Мы тут")
        res = func()
        return res
    return wrapper
@deco
def my_good_func():
    """Очень важный докстринг"""
    return 0
print(my_good_func())
print(my_good_func.__doc__)         # до wraps вывод - Менее..
print(my_good_func.__name__)        # до wraps вывод - wrapper

Декоратор для асинхронной функции. В нём вместо функции подключается корутина.

from typing import Coroutine
import asyncio
def deco(coroutine: Coroutine):
    async def wrapper(*args, **kwargs):
        res = await coroutine(*args, **kwargs)
        return res
    return wrapper
@deco
async def my_async_func():
    print("Начинаем")
    await asyncio.sleep(0.42)
    print("Поспали...")
    return 0
asyncio.run(my_async_func())
# my_async_func = deco(my_async_func)

=== Генераторы и итераторы ===

Итериторивать - перебирать. 

Генератор - это функция, которая возвращает итерируемое множество элементов по одному за раз. Генератор запускается через for. Когда функция доходит до оператора yield, генератор отдаёт управление и значение обратно в for.

from random import randint
def lottery():
    for i in range(10):
        yield randint(1, 100)
for rand_num in lottery():
    print(f"Ваше число: {rand_num}")



=== Функции map() и filter() ===

Списки можно перебирать. Например, есть список из двух строк. Создаётся пустой список. Циклом for итерируется каждый элемент исходного списка и к каждой строке применяется функция title() - делает первую букву заглавной; далее каждая строка добавляется в новый список.

names = ['first', 'second']
uppered_names = []
for name in names:
    uppered_names.append(name.title())
print(uppered_names)

Тоже самое можно сделать через однострочник через функцию map() и list(). Суть map() - применить функцию к каждому элементу коллекции. Синтаксис map() - map(ссылка на функцию, итерируемый объект).

names = ["super", "person"]
title_names = list(map(str.title, names))
print(title_names)

Функция filter() нужна, чтобы удалять элементы по ответу проверяющей функции. Например, есть список чисел от 9 до 1. Можно написать функции проверки чисел на чётность и если текущее число будет нечётными, то оно остаётся - отфильтровывается, а иначе удаляется.

numbers = [9, 8, 7, 6, 5, 4, 3, 2, 1]
def odd(number):
    return number % 2
odds = list(filter(odd, numbers))
print(odds)         # оставить только нечётные

Интересно, что если вместо filter() применить здесь map(), то в список odds попадут результаты выполнения функции odd() на всех значениях.

Можно первым аргументом передать в filter() лямбда-функцию, чтобы сделать однострочник.

numbers = [1, 3, 5, 7, 9, 10, 12, 14, 16, 19]
evens = list(filter(lambda num: not num % 2, numbers))
print(evens)        # оставить только чётные

Кроме конструктора есть ещё деструктор __del__ - функция, которая тоже вызывается при создании объекта и нужна для освобождения определённых ресурсов

class Person:
	def __init__(self):
		print("Я создан")
	def __del__(self):
		print("А теперь что-то удалим...")

def create_my_profile():
	new_me = Person()

create_my_profile()
print("The End")

Тут специально объект создан через функцию, чтобы показать, что он будет существовать только в рамках этой функции. И после завершения функции create_my_profile будет выведено сообщение The End.

По-умолчанию свойства и методы объекта общедоступны. То есть извне можно менять их значения у объекта. Чтобы убрать возможность изменения атрибутов, к имени атрибута можно дописать два подчёрчикания __

class Gamer:
	field: int					# поле
    # поле field может быть доступно без создания объекта

	def __init__(self, name, age, status):
		self.__name = name    			# protected
		self._age = age	      			# private
		self.status = status			# public

	def print_gamer(self):
		print(self.__name)

new_gamer = Gamer("Neo")
new_gamer.__name = "Smith"
new_gamer.print_gamer()

В таком случае изменения атрибута не произойдёт. При желании можно обратиться к приватному атрибуту и изменить его через атрибут класса, например _Gamer__name , но тогда надо угадать правильное имя такого атрибута.

Сеттеры и геттеры - это специальные методы доступа для установки новых значений атрибутов и получения их значений.

***

class Person:
    def __init__(self, name, age):
        self.__name = name
        self.__age = age
    def set_age(self, age):
        if 0 < age < 120:
            self.__age = age
        else:
            print("Ошибка в возрасте")
    def get_name(self):
        return self.__name
    def get_age(self):
        return self.__age
    def print_person(self):
        print(f"Имя: {self.__name}\nВозраст: {self.__age}")
tom = Person("Neo", 22)
tom.print_person()
tom.set_age(-12)
tom.set_age(28)
tom.print_person()

Другой способ - это свойства (@property и @имя_свойства_геттера.setter).

class Person:
    def __init__(self, name, age):
        self.__name = name
        self.__age = age
    
    @property               # свойство-геттер
    def age(self):
        return self.__age
    @age.setter             # свойство-сеттер
    def age(self, age):
        if 0 < age < 120:
            self.__age = age
        else:
            print("Ошибочный возраст")
    
    @property
    def name(self):
        return self.__name
    
    def print_person(self):
        print(f"Имя: {self.__name}, возраст: {self.__age}")

neo = Person("Neo", 18)
neo.print_person()
neo.age = -28
print(neo.age)
neo.age = 28
neo.print_person()

При этом можно геттер использовать отдельно, без сеттера. Чтобы не менять значение, а только получать.



=== Наследование ===

Один из принципов ООП, который позволяет сокращать код, чтобы быстро создавать новые классы (подклассы) от родительских. Например есть класс Person, в которым есть private атрибут __name, геттер и метод отображения данных.

class Person:
    def __init__(self, name):
        self.__name = name
    @property
    def name(self):
        return self.__name
    def display_info(self):
        print(f"Name: {self.__name} ")
neo = Person("Neo")
neo.display_info()

Можно сделать дочерний класс Worker, который унаследует все public атрибуты и методы родительского класса Person.

class Worker(Person):
    def work(self):
        print("Готов вкалывать!")
human = Worker("Worker 1")
human.display_info()
human.work()

Ещё можно делать множественное наследование, когда подкласс наследует атрибуты и методы нескольких классов-родителей. Например, можно сделать РаботающегоСтудента.

class Worker():
    def work(self):
        print("I am working!")
class Student():
    def study(self):
        print("I am studing!")
class WorkingStudent(Worker, Student):
    pass
agent = WorkingStudent()
agent.work()
agent.study()

Оператор pass - это заглушка, с которой можно сделать шаблон класса.

Если унаследовать от нескольких классно атрибуты или методы с одинаковыми именами, то выполняться будут первые из них по цепочке наследования.

class Employee:
    def do(self):
        print("Employee works") 
class Student:
    def do(self):
        print("Student studies")
class WorkingStudent(Student, Employee):
    pass
tom = WorkingStudent()
tom.do()                                    # Student studies

Технически можно смотреть очередность выполнения атрибутов, методов от супер-классов через обращение в классе-наследнике к атрибуту __mro__ или вызов метода mro().

print(WorkingStudent.__mro__)
print(WorkingStudent.mro())



=== Переопределение функционала базового класса ===

Класс-наследник может не только перенимать атрибуты и методы класса-родителя, но и дополнять их. Например, есть класс Person и подкласс Worker. В классе будет конструктор с атрибутом имени, а в подклассе с именем и компанией. В классе будет метод вывода данных объекта, а в подклассе добавляется вывод компании к прочим данным.

class Person:
    def __init__(self, name):
        self.__name = name
    @property
    def name(self):
        return self.__name
    def show_info(self):
        print(f"Меня зовут: {self.__name}")
class Worker(Person):
    def __init__(self, name, comp):
        super().__init__(name)
        self.__comp = comp
    def show_info(self):
        super().show_info()
        print(f"Я работаю в: {self.__comp}")
neo = Worker("Neo", "Microsoft")
neo.show_info()

Объекты можно проверять на тип функцией isinstance().

class Person:
    def __init__(self, name):
        self.__name = name
    @property
    def name(self):
        return self.__name
    def do_nothing(self):
        print(f"Меня зовут {self.name} и я ничего не делаю")
class Worker(Person):
    def work(self):
        print(f"Меня зовут {self.name} и я ГОТОВ ВКАЛЫВАТЬ")
class Student(Person):
    def study(self):
        print(f"Меня зовут {self.name} и я пошёл учиться")
def what_to_do(npc):
    if isinstance(npc, Worker):
        npc.work()
    elif isinstance(npc, Student):
        npc.study()
    elif isinstance(npc, Person):
        npc.do_nothing()
neo = Person("Neo")
smith = Worker("Smith")
vim = Student("Vim")
what_to_do(neo)
what_to_do(smith)
what_to_do(vim)

Такая реализация будет работать, но не совсем в рамках принципа ООП полиморфизм. Здесь лучше всем классам задать единый метод (например `do`), чтобы объект каждого класса сам определял, что ему запускать. Тогда проверка на isinstance будет не нужна.



=== Атрибуты классов и статические методы ===

Атрибуты можно задать не только объекту, но и классу.

class Person:
    type = "Person"
    def __init__(self, name):
        self.name = name

tom = Person("Tom")
bob = Person("Bob")

print(f"""{tom.name} {tom.type}
{bob.name} {bob.type}
{Person.type}""")

Person.type = "Class Person"

print(f"""{tom.name} {tom.type}
{bob.name} {bob.type}
{Person.type}""")

Атрибуты класса можно использовать как значения по-умолчанию для атрибутов объекта.

class Animal:
    default_name = "Cute Dog"
    def __init__(self, name):
        if name:
            self.name = name
        else:
            self.name = Animal.default_name

rex = Animal("")
print(rex.name)

Если в классе не задан атрибут объекта, но к этому атрибуту идёт обращение в конструкторе, то будет проверка есть ли такой атрибут в классе и если есть, то будет передано значение атрибута класса.

class Worker:
    name = "Good Man"

    def show_info(self):
        print(self.name)

work_man = Worker()
work_man.show_info()
work_man.name = "Tim"
work_man.show_info()

В классе могут быть статические методы. Это такие методы, которые применяются в общем ко всему классу. Например, есть класс Person, в нём есть приватный атрибут __type и статический метод print_type(), который будет выводить атрибут класса. Есть два способа вызвать статический метод класса: через Имя_класса.print_type() или через имя_объекта.print_type()

class Person:
    __type = "Person"

    @staticmethod
    def print_type():
        print(Person.__type)

Person.print_type()

bob = Person()
bob.print_type()



=== Класс object. Строковое представление объекта ===

У всех классов есть супер-класс object. У этого супер-класса есть метод __str__ который возвращает строковое представление объекта. Если вывести в консоль сам объект, то туда попадёт строка вида `<__main__.Worker object at 0x0000017528078EC0>`. Можно переопределить метод __str__ для объектов в классе, чтобы выводить более информативные данные о самих объектах.

class Worker:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def display_info(self):
        print(self)

    def __str__(self):
        return f"Я {self.name} и мне {self.age} лет"

neo = Worker("Neo", 28)
print(neo)



=== Перегрузка операторов ===

В модуле operator есть ряд встроенных операторов, типа сложения, равенства и проверки на True/False. Для классов эти операторы можно переопределять. Например, нужно сделать сложение значений объектов классов. Для этого в классе переопределяется встроенный метод __add__(), который принимает атрибут первого объекта и атрибут второго объекта, а возвращает создание нового объекта этого класса со значением суммы значений первых двух объектов.

class Counter:
    def __init__(self, value):
        self.value = value
    
    def __add__(self, other):
        return Counter(self.value + other.value)

c1 = Counter(10)
c2 = Counter(15)
c3 = c1.value + c2.value        # 10 + 15
print(c3)                       # Вывод: 25

Ещё можно изобразить оператор сложения в классе иначе. Например, нужно складывать value объекта и любой другой тип (другое значение). При этом не обязательно возвращать создание нового объекта, можно вернуть только результат вычисления.

class Counter:
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return self.value + other

c1 = Counter(2)
c2 = 4
result = c1 + c2        # 2 + 4
print(result)           # Вывод: 6



=== Абстрактные классы и методы ===

В реальном мире не все категории сущностей реальны. Например, собаки и кошки есть, а `животное` - это абстракция; audi и toyota, как марки машин, а `машина` - это абстракция. В питоне инструменты для создания абстрактных классов помещены в модуль `abc`. Для создания такого класса нужен метод abc.ABC и аннотация @abstractmethod для абстрактного метода. Например, нужно определить абстрактный класс `Животное`.

import abc
class Animal(abc.ABC):
    pass

Здесь не будут создаваться объекты класса `Животное`, а сам класс будет нужен для обозначения которых методов для классов-наследников. Например, разные животные 'говорят' по разному, но у всех будет один метод `speak`, который будет абстрактным в супер-классе.

import abc
class Animal(abc.ABC):
    @abc.abstractmethod
    def speak(self):
        pass

Классы-наследники должны реализовать абстрактные методы абстрактного класса-родителя.

import abc

class Animal(abc.ABC):
    @abc.abstractmethod
    def speak(self):
        pass

class Dog(Animal):
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"I am dog with name: {self.name}"

class Cat(Animal):
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"I am cat with name: {self.name}"
    
bobik = Dog("Bobik")
barsik = Cat("Barsik")
print(f"Собака: {bobik.speak()}\nКошка: {barsik.speak()}")

Абстрактные классы могут задавать конструкторы, атрибуты и неабстрактные методы, которые будут перенимать классы-наследники.



=== Конструкция try...except...finally ===

Есть два типа ошибок: синтаксические (syntax) и ошибки выполнения (runtime). Синтаксические ошибки обычно видно при написании кода прямо в IDE. В процессе выполнения программы появляются ошибки выполнения, которые называют исключениями. Так, например, при попытке преобразовать строку из букв в число, будет ошибка ValueError.

text = "hello"
number = int(text)
print(number)

Чтобы не прерывать программу на таких ошибках придумана конструкция try...except. Она работает так: если при попытке запустить код после `try` была брошена какая-то ошибка, то нужно выполнить специальные инструкции после `except`.

try:
    text = float("42.0")
    number = int(text)
    print(number)
except ValueError:
    print("Получены ошибочные данные")      # Если в int передать str
print("Конец программы")

Чтобы после конструкции try...except выполнить дополнительный код, есть оператор finally. Обычно он нужен, чтобы освободить ресурсы, например закрыть файл.

try:
    hi = "42"
    print(int(hi))
except:
    print("Ошибка данных")
finally:
    print("+ дополнительный код")
print("The end")

Если finally будет без except, то при исключении он не выполнится, потому что finally не обрабатывает исключения, а лишь запускает дополнительный код.



=== except и обработка разных типов исключений ===

Базовые типы исключений в питоне: BaseException, Exception, ArithmeticError (ZeroDivisionError), BufferError, LookupError (IndexError, KeyError).

Можно указывать несколько исключений в блоке try-except, тогда будет выбрано подходящее.

try:
    num1 = int(input("Введите число 1: "))
    num2 = int(input("Введите число 2: "))
    result = num1 / num2
except ValueError:
    print("Преобразование прошло неудачно")
except ZeroDivisionError:
    print("Попытка деления числа на ноль")
except BaseException:
    print("Общее исключение")
print("Завершение программы")

Или можно указать несколько исключений в скобках, тогда при появлении одного из них будут выполнены нужные инструкции.

try:
    num1 = int(input("Введите число 1: "))
    num2 = int(input("Введите число 2: "))
    result = num1 / num2
except (ValueError, ZeroDivisionError, BaseException):
    print("Ошибка ввода данных")

Можно получать информацию об исключении через ключевое слово `as` и записью в переменную.

try:
    num1 = int(input("Введите число 1: "))
    num2 = int(input("Введите число 2: "))
    result = num1 / num2
except ZeroDivisionError as e:
    print(f"Ошибка ввода данных, прочитайте: \"{e}\"")



=== Генерация исключений и создание своих типов исключений ===

Можно создавать свои исключения через ключевое слово `raise` и создание объекта типа `Exception`.

try:
    age = int(input("Введите возраст: "))
    if not 1 <= age <= 120:
        raise Exception("Ошибочный возраст")
    print(f"Вам: {age} лет")
except ValueError:
    print("Ошибка данных")
except Exception as e:
    print(e)
