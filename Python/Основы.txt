=== Область видимости переменных ===

Переменные могут быть глобальными и локальными. Глобальная - это когда например переменная объявлена наверху, а через функцию к ней идёт обращение. А локальная - это когда переменная объявлена внутри функции. Если определить локальную переменную с тем же именем, которое уже есть у глобальной, то в рамках этой функции будет использоваться локальная переменная. Чтобы поменять значение глобальной переменной изнутри функции используется ключевое слово `global`

name = "Neo"
def func():
	global name
	name = "Smith"
	print(name)			# Вывод: Smith
func()
print(name)				# Вывод: Smith

Ещё есть ключевое слово `nonlocal`, с которым можно изменить значение в нелокальной переменной выше уровнем.

def outer():
    num = 8
    def inner():
        nonlocal num
        num = 42
        print(num)
    inner()			# Вывод: 42
    print(num)		# Вывод: 42
outer()

Это такая большая функция, которая включает в себя локальное окружение из переменных и внутренних функций. Например, можно создать функцию func, которая при вызове будет передавать аргумент в параметр для внутренней функции и возвращать саму внутреннюю функцию:

def func(first):
	def second(second):
		return first + second
	return second
sum = func(1)
print(sum(10))		# Вывод: 11
print(sum(20))		# Вывод: 21

А при записи возращенной функции в переменную и вызове её, можно будет передать второй аргумент для sum. Таким образом в переменной лежит ссылка на замыкание (функция + окружение).



=== Декораторы ===

Это такая функция-обёртка, которой можно внести изменения в рабочую функцию не меняя её саму. Например, есть функция hello, которая выводит одну строку. Над ней можно объявить декоратор decorator_func. И чтобы применить декоратор в исходной функции, над ней можно написать название декоратора через 'собачку'

def decorator_func(input_func):
    def new_func():
        print('*' * 24)
        input_func()
        print('*' * 24)
    return new_func

@decorator_func
def hello(): print("Hello, World!")
hello()

Так при вызове исходной функции будет вызван декоратор

Ещё декораторы используют для работы с параметрами функции. Например, есть некая функция суммы двух чисел. Можно при её вызове через декоратор внести в неё изменения, получив отдельно переданные в неё аргументы *args, например 2 и -4, и если сумма этих чисел будет меньше 0, то результатом функция вернёт другое значение, например 0, а не сам результат выполнения функции sum

def modify_sum(original_func):
	def result_func(*args):
		result = original_func(*args)
		if result < 0:
			print(f"увы, тут result: {result}") # Вывод: -2
			return 0
		else:
			return result
	return result_func

@modify_sum
def sum(a, b):
	return a + b

print(sum(2, -4))   # Вывод: 0
print(sum(4, 8))    # Вывод: 12

Вообще есть два способа вызвать декоратор. Первый способ - через 'собачку' перед именем исходной функции. Второй спосок - через передачу исходной функции аргументом в декоратор. Например:

def deco():
	pass

def my_func():
	return 42

my_func = deco(my_func)		# Переопределение исходной функции, декоратор принимает функцию

Декоратор - это паттерн программирования, который позволят добавлять в функцию новый функционал не изменяя саму функцию.

Зачем декораторы? Чтобы считать время выполнения функции. Чтобы логировать параметры функции. Чтобы повторить функцию после ошибки. Чтобы поставить ограничение на частоту запусков функции.

На продакшене обычно использую типизацию. Из модуля typing импортируется объект Callable, который можно вызывать.

from typing import Callable
def empty_deco(func: Callable):
	def wrapper():				# обёртка
		res = func()			# вызов самой функции
		return res				# возврат результата вызываемой функции
	return wrapper				# возврат другой функции после переопределения
def my_func():
    return 42
my_func = empty_deco(my_func)
print(my_func())

Простейший пример применения декоратора - это декоратор подсчёта времени выполнения исходной функции.

from typing import Callable
import time
def timer_deco(func: Callable):
    def wrapper():
        start = time.time()
        res = func()
        end = time.time()
        print(f"Функция отработала за: {end-start}")
        return res
    return wrapper
@timer_deco
def hello_func():
    print("Начинаем спать")
    time.sleep(1.5)
    print("Закончили спать")
    return "Hello_World"
print(hello_func())

Также можно сделать декоратор подсчёта времени для функции с параметром. Здесь в качестве параметра будет передаваться время сна. Переметры передаются через 'арги' и 'кварги' в обёртку.

from typing import Callable
import time
def param_timer_deco(func: Callable):
    def wrapper(*args, **kwargs):
        start = time.time()
        res = func(*args, **kwargs)
        end = time.time()
        print(f"Функция работала: {end-start}")
        return res
    return wrapper
@param_timer_deco
def my_func(sleep_time: int):
    time.sleep(sleep_time)
    return 42+42
print(my_func(2))

Декоратор с параметром. Допустим, мы хотим написать декоратор, который будет лимитировать количество вызовов исходной функции. Для этого в декораторе добавляется ещё один уровень вложенности - дополнительная обёртка для параметра лимита.

from typing import Callable
import time
def limit_deco(limit: int):
    def inner(func: Callable):
        def wrapper():
            nonlocal limit
            if 0 == limit:
                print("Лимит исчерпан")
                return 404
            limit -= 1
            start_time = time.time()
            res = func()
            end_time = time.time()
            print(f"Время исполнения: {end_time-start_time}")
            return res
        return wrapper
    return inner
@limit_deco(2)
def my_func():
    time.sleep(.2)
    return 200
print(my_func())
print(my_func())
print(my_func())

При использовании декоратора не так просто вывести докстринг исходной функции. Для получения докстринга и имени исходной функции подключается отдельный декоратор wraps из модуля functools. Для включения этот wraps подключается через 'собачку' перед объявлением функции обёртки внутри декоратора.
from typing import Callable
from functools import wraps
def deco(func: Callable):
    @wraps(func)
    def wrapper():
        """Менее важный докстринг"""
        print("Мы тут")
        res = func()
        return res
    return wrapper
@deco
def my_good_func():
    """Очень важный докстринг"""
    return 0
print(my_good_func())
print(my_good_func.__doc__)         # до wraps вывод - Менее..
print(my_good_func.__name__)        # до wraps вывод - wrapper

Декоратор для асинхронной функции. В нём вместо функции подключается корутина.

from typing import Coroutine
import asyncio
def deco(coroutine: Coroutine):
    async def wrapper(*args, **kwargs):
        res = await coroutine(*args, **kwargs)
        return res
    return wrapper
@deco
async def my_async_func():
    print("Начинаем")
    await asyncio.sleep(0.42)
    print("Поспали...")
    return 0
asyncio.run(my_async_func())
# my_async_func = deco(my_async_func)

По-умолчанию свойства и методы объекта общедоступны. То есть извне можно менять их значения у объекта. Чтобы убрать возможность изменения атрибутов, к имени атрибута можно дописать два подчёрчикания __

class Gamer:
	field: int					# поле
    # поле field может быть доступно без создания объекта

	def __init__(self, name, age, status):
		self.__name = name    			# protected
		self._age = age	      			# private
		self.status = status			# public

	def print_gamer(self):
		print(self.__name)

new_gamer = Gamer("Neo")
new_gamer.__name = "Smith"
new_gamer.print_gamer()

В таком случае изменения атрибута не произойдёт. При желании можно обратиться к приватному атрибуту и изменить его через атрибут класса, например _Gamer__name , но тогда надо угадать правильное имя такого атрибута.

Сеттеры и геттеры - это специальные методы доступа для установки новых значений атрибутов и получения их значений.

***
@property, @staticmethod, @classmethod, @dataclass, @getter, @setter, *@deletter
***

class Person:
    def __init__(self, name, age):
        self.__name = name
        self.__age = age
    def set_age(self, age):
        if 0 < age < 120:
            self.__age = age
        else:
            print("Ошибка в возрасте")
    def get_name(self):
        return self.__name
    def get_age(self):
        return self.__age
    def print_person(self):
        print(f"Имя: {self.__name}\nВозраст: {self.__age}")
tom = Person("Neo", 22)
tom.print_person()
tom.set_age(-12)
tom.set_age(28)
tom.print_person()

Другой способ - это свойства (@property и @имя_свойства_геттера.setter).

class Person:
    def __init__(self, name, age):
        self.__name = name
        self.__age = age
    
    @property               # свойство-геттер
    def age(self):
        return self.__age
    @age.setter             # свойство-сеттер
    def age(self, age):
        if 0 < age < 120:
            self.__age = age
        else:
            print("Ошибочный возраст")
    
    @property
    def name(self):
        return self.__name
    
    def print_person(self):
        print(f"Имя: {self.__name}, возраст: {self.__age}")

neo = Person("Neo", 18)
neo.print_person()
neo.age = -28
print(neo.age)
neo.age = 28
neo.print_person()

При этом можно геттер использовать отдельно, без сеттера. Чтобы не менять значение, а только получать.
